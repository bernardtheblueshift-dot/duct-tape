   - Extracts `tenant_id` from payload

Create FastAPI router with all auth endpoints:

**Create backend/app/api/v1/auth.py:**

Use `APIRouter(prefix="/api/v1/auth", tags=["auth"])`

**Endpoint 1: POST /register**
- Request: RegisterRequest (email, password, company_name)
- Check if email already exists (query User table)
- If exists, raise HTTPException 400 "Email already registered"
- Create Tenant with name=company_name
- Add to session and flush to get tenant.id
- Create User with email, hashed_password=hash_password(password), tenant_id=tenant.id, role=UserRole.ADMIN, is_active=False
- Add to session and flush to get user.id
- Create VerificationToken using VerificationToken.create_for_user(user.id)
- Add to session and commit
- Call send_verification_email.delay(user.email, verification_token.token)
- Return MessageResponse(message="Verification email sent")

**Endpoint 2: POST /verify-email**
- Request: VerifyEmailRequest (token)
- Query VerificationToken by token
- If not found or expired (expires_at < utcnow), raise HTTPException 400
- Get User by token.user_id
- Set user.is_active = True
- Delete verification token
- Commit
- Return MessageResponse(message="Email verified")

**Endpoint 3: POST /login**
- Request: LoginRequest (email, password) OR OAuth2PasswordRequestForm
- Response parameter for setting cookies
- Query User by email (no tenant filter - login happens before tenant context)
- If not found, raise HTTPException 401 "Invalid credentials"
- Verify password using verify_password
- If incorrect, raise HTTPException 401 "Invalid credentials"
- If not user.is_active, raise HTTPException 403 "Email not verified"
- Create access_token = create_access_token(str(user.id), str(user.tenant_id), user.role.value)
- Create refresh_token = create_refresh_token(str(user.id))
- Set cookies on response:
  - access_token: httponly=True, secure=True, samesite="lax", max_age=15*60
  - refresh_token: httponly=True, secure=True, samesite="lax", max_age=7*24*60*60
- Return MessageResponse(message="Login successful")

**Endpoint 4: POST /refresh**
- Cookie parameter: refresh_token
- Response parameter for setting new cookie
- Decode refresh token (use jwt.decode directly, checking type="refresh")
- Extract user_id from "sub" claim
- Query User by id
- Create new access_token
- Set access_token cookie on response
- Return MessageResponse(message="Token refreshed")

**Endpoint 5: POST /reset-password-request**
- Request: PasswordResetRequest (email)
- Query User by email
- If not found, still return success (don't leak user existence)
- Create PasswordResetToken using PasswordResetToken.create_for_user(user.id)
- Call send_password_reset_email.delay(user.email, token.token)
- Return MessageResponse(message="Password reset email sent if account exists")
