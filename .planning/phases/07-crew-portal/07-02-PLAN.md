    **2. Add endpoints to backend/app/api/v1/portal.py:**

    **GET /assignments** (require_active):
    - Look up CrewProfile for current_user.id. 404 if not found.
    - Query CrewAssignment JOIN Job WHERE crew_id = profile.id. Order by Job.scheduled_start ASC NULLS LAST.
    - Return list[PortalAssignmentDetail].

    **POST /assignments/{assignment_id}/confirm** (require_active):
    - Look up CrewProfile for current_user.id. 404 if not found.
    - Look up CrewAssignment by assignment_id. 404 if not found.
    - Verify assignment.crew_id == profile.id. 403 "Cannot act on another crew member's assignment" if not.
    - Check ASSIGNMENT_TRANSITIONS[assignment.status] contains CONFIRMED. 400 "Invalid transition: {current} -> confirmed" if not.
    - Set assignment.status = AssignmentState.CONFIRMED. Commit. Return CrewAssignmentResponse.

    **POST /assignments/{assignment_id}/decline** (require_active):
    - Same ownership check as confirm.
    - Accept PortalDeclineRequest body (optional declined_reason).
    - Check ASSIGNMENT_TRANSITIONS allows DECLINED from current state. 400 if not.
    - Set assignment.status = AssignmentState.DECLINED, assignment.declined_reason = body.declined_reason. Commit. Return CrewAssignmentResponse.

    **GET /profile** (require_active):
    - Look up CrewProfile for current_user.id. 404 "Crew profile not found" if not found.
    - Return CrewProfileResponse.

    **PATCH /profile** (require_active):
    - Look up CrewProfile for current_user.id. 404 if not found.
    - Accept PortalProfileUpdate body.
    - Only apply phone and bio from update_data = body.model_dump(exclude_unset=True). Do NOT apply any other fields even if somehow present.
    - Commit. Return CrewProfileResponse.

    **PUT /availability** (require_active):
    - Look up CrewProfile for current_user.id. 404 if not found.
    - Accept List[AvailabilityPatternCreate] body.
    - Delete existing AvailabilityPattern WHERE crew_id = profile.id.
    - Insert new patterns with crew_id=profile.id, tenant_id=tenant_id.
    - Commit. Return List[AvailabilityPatternResponse].

    **GET /availability** (require_active):
    - Look up CrewProfile for current_user.id. 404 if not found.
    - Query AvailabilityPattern WHERE crew_id = profile.id ORDER BY day_of_week ASC.
    - Return List[AvailabilityPatternResponse].

    **3. Write tests in backend/tests/test_portal_actions.py** covering all behaviors above. Use existing conftest.py patterns for async fixtures.
