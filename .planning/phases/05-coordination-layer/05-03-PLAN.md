    **backend/app/api/v1/tasks.py:**
    Create router with prefix="/api/v1/jobs/{job_id}/tasks", tags=["tasks"].

    Endpoints:

    1. POST "/" — create_task (admin only)
       - current_user: User = Depends(require_admin)
       - Verify job exists: select(Job).where(Job.id == job_id), 404 if not
       - If task_data.assignee_id is set, verify CrewProfile exists: select(CrewProfile).where(CrewProfile.id == task_data.assignee_id), 404 "Assignee not found" if not
       - If task_data.message_id is set, verify Message exists in same job: select(Message).where(Message.id == task_data.message_id, Message.job_id == job_id), 404 "Message not found" if not
       - Create Task(**task_data.model_dump(), job_id=job_id, tenant_id=tenant_id)
       - Return TaskResponse, status 201

    2. GET "/" — list_tasks
       - current_user: User = Depends(require_active) (both admin and crew can read)
       - Query params: status (TaskStatus | None), assignee_id (UUID | None)
       - Base query: select(Task).where(Task.job_id == job_id)
       - If status filter: .where(Task.status == status)
       - If assignee_id filter: .where(Task.assignee_id == assignee_id)
       - Order by: Task.priority.desc() (urgent first), then Task.created_at.asc()
       - Note: TaskPriority enum ordering — define a manual sort: URGENT > HIGH > MEDIUM > LOW. Use case() expression or order by raw SQL: ORDER BY CASE priority WHEN 'urgent' THEN 4 WHEN 'high' THEN 3 WHEN 'medium' THEN 2 WHEN 'low' THEN 1 END DESC, created_at ASC
       - Return list of TaskResponse

    3. GET "/{task_id}" — get_task
       - current_user: User = Depends(require_active)
       - select(Task).where(Task.id == task_id, Task.job_id == job_id), 404 if not found
       - Return TaskResponse

    4. PATCH "/{task_id}" — update_task (admin only)
       - current_user: User = Depends(require_admin)
       - Find task, 404 if not found
       - If update includes assignee_id, verify CrewProfile exists
       - update_data = task_update.model_dump(exclude_unset=True)
       - Apply updates, commit, refresh
       - Return TaskResponse

    5. POST "/{task_id}/status" — update_task_status (admin OR assigned crew)
       - current_user: User = Depends(require_active) — NOT require_admin
       - Find task, 404 if not found
       - Permission check: if current_user.role != UserRole.ADMIN, then verify task.assignee_id matches current user's crew profile. Query CrewProfile by user_id: select(CrewProfile).where(CrewProfile.user_id == current_user.id). If no profile or profile.id != task.assignee_id, raise 403 "Not authorized to update this task"
       - Import can_transition from app.core.task_state
       - Validate: if not can_transition(task.status, status_update.status), raise 400 f"Invalid transition: {task.status.value} -> {status_update.status.value}"
       - Update task.status, commit, refresh
       - Return TaskResponse

    6. DELETE "/{task_id}" — delete_task (admin only)
       - current_user: User = Depends(require_admin)
       - Find task, 404 if not found
       - Hard delete, commit
       - Return 204
