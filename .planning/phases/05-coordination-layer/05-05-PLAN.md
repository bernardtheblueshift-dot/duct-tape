    Add imports at top:
    ```python
    from sqlalchemy import func
    from datetime import datetime, timezone
    from app.models import Message, Task, TaskStatus, JobFile
    from app.schemas.job import CoordinationSummary, MessageSummary, FileSummary
    ```

    **Create helper function build_coordination_summary(job_id, db) -> CoordinationSummary:**

    This function takes a single job_id and db session, returns a CoordinationSummary:

    1. Message count: select(func.count(Message.id)).where(Message.job_id == job_id)
    2. Recent 3 messages: select(Message).where(Message.job_id == job_id).order_by(Message.created_at.desc()).limit(3)
    3. Task counts by status: select(Task.status, func.count(Task.id)).where(Task.job_id == job_id).group_by(Task.status)
    4. Overdue count: select(func.count(Task.id)).where(Task.job_id == job_id, Task.deadline < datetime.now(timezone.utc), Task.status != TaskStatus.DONE)
    5. File count: select(func.count(JobFile.id)).where(JobFile.job_id == job_id)
    6. Recent 3 files: select(JobFile).where(JobFile.job_id == job_id).order_by(JobFile.created_at.desc()).limit(3)

    Return CoordinationSummary with all fields populated.

    **Create helper function batch_coordination_summaries(job_ids, db) -> dict[UUID, CoordinationSummary]:**

    For the list endpoint, batch queries to avoid N+1:

    1. Message counts per job: select(Message.job_id, func.count(Message.id)).where(Message.job_id.in_(job_ids)).group_by(Message.job_id)
    2. Task counts per job per status: select(Task.job_id, Task.status, func.count(Task.id)).where(Task.job_id.in_(job_ids)).group_by(Task.job_id, Task.status)
    3. Overdue counts per job: select(Task.job_id, func.count(Task.id)).where(Task.job_id.in_(job_ids), Task.deadline < datetime.now(timezone.utc), Task.status != TaskStatus.DONE).group_by(Task.job_id)
    4. File counts per job: select(JobFile.job_id, func.count(JobFile.id)).where(JobFile.job_id.in_(job_ids)).group_by(JobFile.job_id)
    5. Skip recent_messages and recent_files in list view (too expensive for N jobs, empty list default is fine for list view — full data only in detail view)

    Build dict: {job_id: CoordinationSummary(...)} for each job_id. Jobs with no messages/tasks/files get default CoordinationSummary().

    **Update get_job endpoint:**
    - After loading job and assignments, call: coordination = await build_coordination_summary(job.id, db)
    - Replace "messages": [], "tasks": [], "files": [] with "coordination": coordination

    **Update list_jobs endpoint:**
    - After batch assignment queries, add: summaries = await batch_coordination_summaries(job_ids, db)
