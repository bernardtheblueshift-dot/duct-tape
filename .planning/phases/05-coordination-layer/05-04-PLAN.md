<objective>
Implement file upload, serving, and management API with tenant-isolated local filesystem storage.

Purpose: Users can upload files (briefs, runsheets, photos, videos, docs) to a job. Files are stored on local filesystem organized by tenant/job with UUID filenames. Server validates MIME types via python-magic for security. Authenticated file serving endpoint returns files with correct Content-Type.

Output: Files REST router (upload, list, get metadata, serve, delete), integration tests.
</objective>

    **backend/app/api/v1/files.py:**
    Create TWO routers:
    - job_files_router with prefix="/api/v1/jobs/{job_id}/files", tags=["files"] — for upload and list (scoped to a job)
    - files_router with prefix="/api/v1/files", tags=["files"] — for serve and delete by file ID (not job-scoped)

    Endpoints on job_files_router:

    1. POST "/" — upload_file
       - Parameters: job_id: UUID, file: UploadFile = File(...), current_user: User = Depends(require_active), tenant_id: str = Depends(get_current_tenant), db: AsyncSession = Depends(get_db)
       - Verify job exists: select(Job).where(Job.id == job_id), 404 if not
       - try: storage_path, mime_type, file_size = await save_upload(str(tenant_id), str(job_id), file)
       - except ValueError as e: raise HTTPException(400, str(e))
       - Create JobFile(job_id=job_id, uploader_id=current_user.id, original_filename=file.filename or "unnamed", storage_path=str(storage_path), mime_type=mime_type, file_size=file_size, tenant_id=tenant_id)
       - db.add, commit, refresh
       - Return FileResponse, status 201

    2. GET "/" — list_files
       - current_user: User = Depends(require_active) (both admin and crew can view)
       - Query: select(JobFile).where(JobFile.job_id == job_id).order_by(JobFile.created_at.desc())
       - Return list of FileResponse

    Endpoints on files_router:

    3. GET "/{file_id}" — get_file_metadata
       - current_user: User = Depends(require_active)
       - tenant_id: str = Depends(get_current_tenant) (RLS ensures tenant match)
       - select(JobFile).where(JobFile.id == file_id), 404 if not found
       - Return FileResponse

    4. GET "/{file_id}/download" — serve_file
       - current_user: User = Depends(require_active)
       - tenant_id: str = Depends(get_current_tenant) (RLS ensures tenant match — no cross-tenant access)
       - select(JobFile).where(JobFile.id == file_id), 404 if not found
       - from fastapi.responses import FileResponse as FastAPIFileResponse
       - Verify file exists on disk: Path(file_record.storage_path).exists(), 404 "File not found on disk" if not
       - Return FastAPIFileResponse(path=file_record.storage_path, media_type=file_record.mime_type, filename=file_record.original_filename)

    5. DELETE "/{file_id}" — delete_file_endpoint (admin only)
       - current_user: User = Depends(require_admin)
       - tenant_id: str = Depends(get_current_tenant)
       - Find file, 404 if not found
       - Delete from DB first: db.delete(file_record), commit
       - Then delete from disk: await delete_file(Path(file_record.storage_path))
       - Return 204
