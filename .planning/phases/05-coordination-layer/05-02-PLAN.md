Implement messaging REST API endpoints and WebSocket real-time delivery for job-scoped threaded messaging.

Output: Messages REST router (POST, GET list, GET single), WebSocket endpoint with subscribe/unsubscribe protocol, integration tests for both.

    1. POST "/" — create_message(job_id: UUID, message_data: MessageCreate, current_user: User = Depends(require_active), tenant_id: str = Depends(get_current_tenant), db: AsyncSession = Depends(get_db))
       - Verify job exists: select(Job).where(Job.id == job_id), 404 if not found
       - If message_data.reply_to_id is set, verify parent message exists in same job, 404 if not
       - If message_data.file_ids is non-empty, validate all file IDs exist and belong to same job: select(JobFile).where(JobFile.id.in_(file_ids), JobFile.job_id == job_id). If count != len(file_ids), raise 400 "Invalid file IDs"
       - Create Message(job_id=job_id, user_id=current_user.id, content=message_data.content, reply_to_id=message_data.reply_to_id, tenant_id=tenant_id)
       - If file_ids: load the JobFile objects and append to message.files relationship
       - commit, refresh
       - Broadcast to WebSocket: await manager.broadcast_to_job(str(job_id), {"type": "message", "data": MessageResponse.model_validate(message).model_dump(mode="json")})
       - Return message with status 201

    2. GET "/" — list_messages(job_id: UUID, search: str | None = None, tenant_id: str = Depends(get_current_tenant), db: AsyncSession = Depends(get_db))
       - Both admin and crew can read messages (use require_active, not require_admin)
       - Base query: select(Message).where(Message.job_id == job_id)
       - If search: add .where(Message.content.ilike(f"%{search}%"))
       - Order by Message.created_at.asc() (oldest first, chat-style)
       - Return list of MessageResponse

    3. GET "/{message_id}" — get_message(job_id: UUID, message_id: UUID, tenant_id: str = Depends(get_current_tenant), db: AsyncSession = Depends(get_db))
       - select(Message).where(Message.id == message_id, Message.job_id == job_id)
       - 404 if not found
       - Return MessageResponse

    **backend/app/api/v1/websocket.py:**
    Create router with no prefix (WebSocket mounted at root).

    Endpoint:
    @router.websocket("/ws")
    async def websocket_endpoint(websocket: WebSocket, token: str):
       - Import decode_access_token from app.core.security. Call it with token param. On HTTPException, close WebSocket with code=1008 (Policy Violation) and return. Extract user_id from payload["sub"].
       - Call await manager.connect(str(user_id), websocket)
       - try/except WebSocketDisconnect block:
         - While loop: await websocket.receive_json()
         - Handle actions: "subscribe" (manager.subscribe), "unsubscribe" (manager.unsubscribe), "ping" (send {"type": "pong"} back)
       - In except block: manager.disconnect(str(user_id))
       - Note: The decode_access_token raises HTTPException, but WebSocket context needs different error handling. Catch Exception broadly around decode_access_token and close with 1008 on any auth failure.
