**backend/app/api/v1/equipment.py:**

Create router: `router = APIRouter(prefix="/api/v1/equipment", tags=["equipment"])`

Endpoints (replicate jobs.py patterns for consistency):

1. `POST /` — Create equipment (admin only)
   - Accepts `EquipmentCreate`, requires `require_admin` + `get_current_tenant` + `get_db`
   - Creates Equipment with `tenant_id=tenant_id`
   - Uses `equipment_data.model_dump()` spread into Equipment constructor
   - Returns 201 with `EquipmentResponse`

2. `GET /` — List/search equipment inventory (any authenticated user)
   - Query params: `search: str | None = None`, `category: str | None = None`, `condition: EquipmentCondition | None = None`
   - Requires `get_current_tenant` + `get_db` (RLS handles tenant filtering)
   - Search: ILIKE on Equipment.name, Equipment.notes (using `or_` pattern)
   - Category filter: `Equipment.category.ilike(f"%{category}%")`
   - Condition filter: `Equipment.condition == condition`
   - Order by `Equipment.name.asc()`
   - Returns `list[EquipmentResponse]`

3. `GET /{equipment_id}` — Get single equipment item
   - Requires `get_current_tenant` + `get_db`
   - Returns 404 if not found

4. `PATCH /{equipment_id}` — Update equipment (admin only)
   - Accepts `EquipmentUpdate`, uses `model_dump(exclude_unset=True)` pattern
   - Returns updated `EquipmentResponse`

5. `DELETE /{equipment_id}` — Delete equipment (admin only)
   - Returns 204 No Content
   - NOTE: Only allow deletion if no active assignments exist (query EquipmentAssignment for this equipment_id, return 409 if any found)

6. `PATCH /{equipment_id}/condition` — Update equipment condition/status (admin only)
   - Accepts body with `condition: EquipmentCondition`
   - Separate endpoint for quick status updates (e.g., marking gear for maintenance)
   - Returns updated `EquipmentResponse`
