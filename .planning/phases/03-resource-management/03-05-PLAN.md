Add 6 new endpoints to the existing crew router in `backend/app/api/v1/crew.py`. Add necessary imports at the top: `CrewRating`, `AvailabilityPattern`, `func` from sqlalchemy, and all new schema types.

**1. `POST /{crew_id}/ratings` — Rate crew (admin only)**
- Accepts `CrewRatingCreate` + path param `job_id: UUID` as query param
- Requires `require_admin`
- Validate crew_id exists (404)
- Validate job_id exists (404)
- Check unique constraint: no existing rating for this crew+job pair (409 "Already rated for this job")
- Create `CrewRating` with `rated_by=current_user.id`, `tenant_id=tenant_id`
- After creating rating, recalculate cached average:
  ```python
  # Query all ratings for this crew
  avg_query = select(func.avg(CrewRating.stars), func.count(CrewRating.id)).where(CrewRating.crew_id == crew_id)
  result = await db.execute(avg_query)
  avg, count = result.one()
  # Update CrewProfile cache
  crew_profile.rating_average = round(float(avg), 2)
  crew_profile.rating_count = count
  ```
- Return 201 with `CrewRatingResponse`

**2. `GET /{crew_id}/ratings` — List ratings for crew (any authenticated user)**
- Returns `list[CrewRatingResponse]` ordered by `created_at` desc
- Supports `limit: int = 20` and `offset: int = 0` query params

**3. `GET /{crew_id}/history` — Crew job history (any authenticated user, covers CREW-06)**
- Query CrewAssignment WHERE crew_id matches, join Job for details
- Return list of `{"job_id": UUID, "job_title": str, "role": str, "status": str, "scheduled_start": datetime, "scheduled_end": datetime}` 
- Create inline response schema `CrewJobHistoryEntry` in the schemas/crew.py (or in the endpoint file)
- Order by Job.scheduled_start desc

**4. `PUT /{crew_id}/availability` — Set availability patterns (crew or admin)**
- Accepts `list[AvailabilityPatternCreate]` (array of day_of_week + is_available pairs)
- Requires `require_active` — crew can set their own, admin can set anyone's
- Validate: if current user is crew role, they can only set their OWN availability (check CrewProfile.user_id == current_user.id, else 403)
- Upsert pattern: delete all existing AvailabilityPattern for this crew_id, then insert new ones
- Return `list[AvailabilityPatternResponse]`

**5. `GET /{crew_id}/availability` — Get availability patterns (any authenticated user)**
- Query AvailabilityPattern WHERE crew_id matches
- Return `list[AvailabilityPatternResponse]` ordered by day_of_week asc

**6. `GET /skills-matrix` — Skills matrix view (any authenticated user)**
- NOTE: This must be defined BEFORE the `/{crew_id}` route to avoid path conflicts
- Query: `SELECT DISTINCT unnest(skills) FROM crew_profiles WHERE archived_at IS NULL` (tenant filtered by RLS)
- Get all non-archived crew profiles
- Build matrix response: for each crew member, map each skill to True/False based on whether skill is in their skills array
- Populate `SkillsMatrixEntry.email` from `User.email` (User model has NO name field -- only email)
- Return `SkillsMatrixResponse` with `skills` (sorted list of all unique skills) and `crew` (list with id, email, skills dict)
