**backend/app/api/v1/assignments.py:**

Create router: `router = APIRouter(prefix="/api/v1/assignments", tags=["assignments"])`

Endpoints:

1. `POST /crew` ‚Äî Assign crew to job (admin only)
   - Accepts `CrewAssignmentCreate`, requires `require_admin` + `get_current_tenant` + `get_db`
   - Validate job exists (404 if not)
   - Validate crew_id exists in crew_profiles (404 if not)
   - Validate crew not already assigned to this job (409 "Crew already assigned to this job")
   - If job has scheduled_start AND scheduled_end:
     - Call `check_crew_conflicts(db, crew_id, job.scheduled_start, job.scheduled_end)`
     - Call `check_crew_availability_patterns(db, crew_id, job.scheduled_start)` to check day-of-week
     - If conflicts found and `force=False`: return 409 with ConflictWarning JSON body containing `{"message": "Crew has scheduling conflicts", "conflicts": [{job_id, job_title, scheduled_start, scheduled_end}]}`
     - If conflicts found and `force=True`: proceed but store `override_reason`
     - If availability pattern blocks and `force=False`: return 409 with `{"message": "Crew is marked unavailable on this day", "conflicts": []}`
   - Create CrewAssignment with `status=AssignmentState.PENDING`, `tenant_id=tenant_id`
   - Return 201 with `CrewAssignmentResponse`

2. `POST /equipment` ‚Äî Assign equipment to job (admin only)
   - Accepts `EquipmentAssignmentCreate`
   - Validate job and equipment exist (404)
   - Validate equipment not already assigned to this job (409)
   - If job has scheduled_start AND scheduled_end:
     - Call `check_equipment_availability(db, equipment_id, job.scheduled_start, job.scheduled_end)`
     - If `available_quantity < quantity_assigned`: return 409 with `{"message": "Insufficient equipment available", "total_quantity": N, "assigned_quantity": N, "available_quantity": N, "requested": N}` ‚Äî HARD BLOCK, no override
   - Create EquipmentAssignment, return 201 with `EquipmentAssignmentResponse`

3. `POST /crew/{assignment_id}/transition` ‚Äî Crew confirms/declines assignment
   - Accepts `AssignmentTransitionRequest`, requires `require_active` (crew can do this, not just admin)
   - Get assignment by id (404 if not found)
   - Validate current user is the crew member (match assignment.crew_id to CrewProfile.user_id) OR is admin
   - Validate transition is allowed using `ASSIGNMENT_TRANSITIONS` dict
   - If declining: store `declined_reason` from request body
   - Update status, return updated `CrewAssignmentResponse`

4. `GET /job/{job_id}/crew` ‚Äî List crew assignments for a job (any authenticated user)
   - Query CrewAssignment WHERE job_id matches
   - Return `list[CrewAssignmentResponse]`

5. `GET /job/{job_id}/equipment` ‚Äî List equipment assignments for a job (any authenticated user)
   - Query EquipmentAssignment WHERE job_id matches
   - Return `list[EquipmentAssignmentResponse]`

6. `DELETE /crew/{assignment_id}` ‚Äî Remove crew assignment (admin only)
   - Get assignment, delete it, return 204

7. `DELETE /equipment/{assignment_id}` ‚Äî Remove equipment assignment (admin only)
   - Get assignment, delete it, return 204
