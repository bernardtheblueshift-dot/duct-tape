**Endpoints to implement:**

1. **POST / (create_job):**
   - Depends: require_admin, get_current_tenant, get_db
   - Create Job with **job_data.model_dump(), tenant_id=tenant_id
   - Add to db, commit, refresh, return
   - Status: 201

2. **GET / (list_jobs):**
   - Query params: search (str | None), state (JobState | None), venue (str | None), start_date (datetime | None), end_date (datetime | None)
   - Depends: get_current_tenant, get_db (no require_admin - crew can view)
   - Build query with dynamic filters (use or_() for search across title/description/venue with ILIKE)
   - Order by Job.scheduled_start.desc()
   - Return list[JobResponse]

3. **GET /{job_id} (get_job):**
   - Depends: get_current_tenant, get_db
   - Select Job where id == job_id
   - Raise 404 if not found
   - Return JobResponse

4. **PATCH /{job_id} (update_job):**
   - Depends: require_admin, get_current_tenant, get_db
   - Select Job where id == job_id, raise 404 if not found
   - Update with job_update.model_dump(exclude_unset=True, exclude={"state"})
   - Commit, refresh, return

5. **DELETE /{job_id} (delete_job):**
   - Depends: require_admin, get_current_tenant, get_db
   - Select Job where id == job_id, raise 404 if not found
   - db.delete(job), commit
   - Status: 204

**Critical pattern (from research pitfall #1):** Every endpoint MUST include `tenant_id: str = Depends(get_current_tenant)` even if not used in function body - this sets RLS context via SET LOCAL.
